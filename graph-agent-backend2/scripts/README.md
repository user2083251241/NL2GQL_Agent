# 测试数据生成器使用说明

## 📖 概述

`generate_test_data.py` 是一个用于向 HugeGraph 数据库批量生成和插入测试数据的脚本。它可以帮助您快速构建大规模的图数据集，用于测试 Agent 的查询能力和性能。

## 🎯 功能特性

- ✅ **自动生成 Schema**：自动创建顶点标签、边标签和属性键
- ✅ **大规模数据生成**：支持自定义节点和边的数量
- ✅ **真实场景模拟**：社交网络 + 知识图谱混合场景
- ✅ **批量插入优化**：高效的批量插入策略
- ✅ **数据验证**：自动验证数据完整性并输出统计信息
- ✅ **安全清空**：可选的数据清空功能（带确认提示）

## 🏗️ 数据模型

### 顶点类型

1. **Person (人物)**
   - 属性：name, age, city, job_title
   - 示例：张伟, 35岁, 北京, 软件工程师

2. **Company (公司)**
   - 属性：name, industry, scale
   - 示例：阿里巴巴, 互联网, 大型

3. **Skill (技能)**
   - 属性：name
   - 示例：Python, Java, 机器学习

### 边关系

1. **knows (认识)**
   - Person → Person
   - 属性：since (认识时间)

2. **works_at (工作于)**
   - Person → Company
   - 属性：since (入职时间)

3. **has_skill (掌握技能)**
   - Person → Skill
   - 属性：proficiency (熟练程度：初级/中级/高级/专家)

4. **invests_in (投资)**
   - Person → Company
   - 属性：amount (投资金额)

## 🚀 快速开始

### 基本用法

```bash
# 使用默认配置生成数据（500个人物，3000条边）
python scripts/generate_test_data.py
```

### 自定义数据规模

```bash
# 生成1000个人物，5000条边
python scripts/generate_test_data.py --vertices 1000 --edges 5000

# 生成2000个人物，10000条边（大规模测试）
python scripts/generate_test_data.py --vertices 2000 --edges 10000
```

### 清空后重新生成

```bash
# ⚠️ 警告：这会删除所有现有数据！
python scripts/generate_test_data.py --clear
```

### 查看帮助

```bash
python scripts/generate_test_data.py --help
```

## 🔧 技术实现细节

### 批量插入策略

- **顶点插入**：每批100个顶点，使用 `g.inject(1).addV()...` 语法
- **边插入**：每批50条边，使用链式 `addE()` 操作
- **事务控制**：每个批次作为一个独立事务执行

### 数据分布

- **works_at 关系**：每个人恰好工作在一家公司（100%覆盖）
- **has_skill 关系**：每人随机掌握2-8个技能（约占边总数的70%）
- **knows 关系**：平均每人认识3个人（社交网络特性）
- **invests_in 关系**：约10%的人有投资行为

### 性能优化

- 使用批量操作减少网络往返
- 避免单点插入，提高吞吐量
- 错误处理和重试机制

## ⚠️ 注意事项

1. **数据清空风险**：使用 `--clear` 参数前请确认是否需要备份数据
2. **内存占用**：大规模数据生成（>5000节点）可能需要较多内存
3. **执行时间**：根据数据规模和网络延迟，可能需要几分钟到十几分钟
4. **唯一性约束**：Person 和 Company 使用 name 作为主键，重复名称会跳过
5. **中文支持**：确保 HugeGraph 配置支持 UTF-8 编码

## 🐛 故障排查

### 问题1：连接失败
```
❌ HugeGraph连接失败: ...
```
**解决方案**：
- 检查 `.env` 文件中的数据库配置
- 确认 HugeGraph 服务正在运行
- 验证网络连接和防火墙设置

### 问题2：Schema创建失败
```
⚠️ 顶点标签已存在或创建失败: Person
```
**解决方案**：
- 这是正常现象，表示Schema已存在
- 如需重新创建，先使用 `--clear` 清空数据

### 问题3：插入速度慢
**解决方案**：
- 检查网络连接质量
- 降低 batch_size 参数（修改代码）
- 确认 HugeGraph 服务器资源充足

### 问题4：数据量不符合预期
**解决方案**：
- 检查生成的统计信息
- 某些边可能因为主键冲突而跳过
- 适当增加目标边数量

```python
    ("has_skill", ["proficiency"]),
    ("invests_in", ["amount"]),
    ("collaborates_with", ["project"])  # 新增
]
```
